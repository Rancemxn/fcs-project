# FCS Render Conformance Assets

The files in this directory are synthetic, project-owned inputs for the FCS Render Profile 1.0
conformance suite. They do not contain copied images, system fonts, community chart artwork, names,
or third-party glyph designs.

- `fcs-test-rgba8.png` encodes the four 2×2 RGBA8 texels declared as `PNG_PIXELS` in the test-only
  asset builder.
- `fcs-test-lossless.webp` encodes the distinct four 2×2 RGBA8 texels declared as `WEBP_PIXELS`.
  The fixed `image 0.25.10` lossless WebP encoder is used only to create the byte-pinned input.
- `fcs-test-font.ttf` is a seven-table, single-face TrueType font generated entirely in this
  repository. Glyph 0 is empty; glyph 1 is a centered four-point square; the Windows Unicode BMP
  format-4 cmap maps only U+0041 `A` to glyph 1. `unitsPerEm` and both advances are 1000.

The authoritative generator and independent restricted decoders are in
`crates/fcs-source/tests/support/fcbc_render_reference_assets.rs`. The ordinary test lane compares
the checked-in bytes with newly generated bytes. Regeneration is an explicit maintainer action via
the `FCS_REGENERATE_RENDER_ASSETS` environment variable; runtime conformance tests never consult
system fonts, external paths, URLs, or platform color services.

Exact byte lengths and SHA-256 digests will be recorded by the I9 nonempty RenderSection artifact
after its enclosing FCBC golden is fixed and independently reviewed. The current pre-I1 manifest
does not claim that absent artifact. All three assets are distributed under the adjacent MIT
license.
